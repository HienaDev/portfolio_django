About My Portfolio Application

Project Overview

This portfolio application showcases my projects, academic journey, and technical skills. It's designed to present my work in an organized and visually appealing manner, highlighting the technologies I've learned and the projects I've developed.

GitHub Repository: https://github.com/HienaDev/portfolio_django.git

Technologies Used

Backend

  • Python 3: Core programming language
  • Django: Web framework for rapid development and clean, pragmatic design
  • SQLite: Lightweight database for development (can be migrated to PostgreSQL in production)

Frontend

  • HTML5: Structure and content
  • CSS3: Styling and animations
  • JavaScript: Interactive elements and particle animations
  • Google Fonts: Cal Sans typography

Deployment & Tools

  • PythonAnywhere: Hosting and deployment
  • Git/GitHub: Version control and code repository
  • Django Template Language: For dynamic content rendering
  • Django ORM: Object-Relational Mapping for database interactions

Application Architecture

The application follows Django's Model-Template-View (MTV) architectural pattern:

MTV Pattern Explained

  • Models (M): Define the data structure, handle data validation, and manage database interactions. In this application, models include Projeto, Aluno, Docente, Disciplina, Tecnologia, and Imagem.
  • Templates (T): Handle the presentation layer, defining how data is displayed to users. Templates use Django Template Language to create dynamic HTML pages.
  • Views (V): Process user requests, interact with models to retrieve data, and pass that data to templates for rendering. They serve as controllers that coordinate the application flow.

Architecture Diagram

Component Interaction

  1. User sends a request to the application (e.g., visiting a project page)
  2. Django's URL dispatcher routes the request to the appropriate view function
  3. The view function processes the request, queries the database using models
  4. Data is passed to the template for rendering
  5. The rendered HTML is sent back to the user's browser

Database Modeling

The application uses a relational database with the following entity relationships:

Entity Relationship Diagram

Key Relationships

  • One-to-One Relationships:
    • Each Projeto is associated with exactly one Disciplina
  • One-to-Many Relationships:
    • A Projeto can have multiple Imagem entities (project images)
  • Many-to-Many Relationships:
    • Projetos can have multiple Alunos (students) and vice versa
    • Projetos can use multiple Tecnologias (technologies) and vice versa
    • Disciplinas can have multiple Docentes (teachers) and vice versa

Models Overview

  • Disciplina (Subject): Represents academic courses with associated teachers
  • Docente (Teacher): Represents faculty members teaching disciplines
  • Aluno (Student): Represents students who participate in projects
  • Tecnologia (Technology): Represents programming languages, frameworks, and tools
  • Projeto (Project): Represents development projects linked to disciplines and students
  • Imagem (Image): Represents visual materials for projects

Application Page Structure

The application follows a hierarchical structure of pages:

Main Navigation Areas

  • Home: Displays a welcome message and overview of projects
  • About: This page - presents technical information about the application
  • Technologies: Lists all technologies with details and associated projects
  • Detail Pages:
    • Project details - Shows comprehensive information about each project
    • Student details - Shows student information and associated projects
    • Teacher details - Shows teacher information and associated disciplines
    • Discipline details - Shows course information, teachers, and projects
    • Technology details - Shows technology information and projects using it
